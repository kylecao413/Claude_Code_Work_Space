# 02c — Fire Protection PRR Fill-out Guide (Fairfax 2021)

**Form:** `Plan_Review_Records/2021/Fire_Plan_Review/fire-plan-review-record-2021.pdf`
**Pair with:** `02a` cheatsheet, `02b` reference, `00_KCY_Deficiency_Log_Template.docx`.

---

## Fill-rule recap (same as Building)

| Mark | When | Comment |
|------|------|---------|
| ☑ Complies | Drawings show compliance | Blank or "OK" |
| ☐ N/A | Doesn't apply | One-line scope reason |
| ☐ + comment | Deficiency | "See Deficiency Log #X" |

**No suggested correction language. No design opinions in Comments column.**

---

## Page 1 — Project / Designer / Peer Reviewer / Code Information

Fill same as Building PRR (project name, address, parcel, permit #, designer + license, KCY peer reviewer + PR #). Code Information block has different shape:
- Check VCC OR VEBC + project type (New / Addition / L1 / L2 / Repair / Change of Occupancy / Moved / Historic / Addition for VEBC)
- Occupancy/Group + Proposed Group(s)
- Mixed-use checkboxes (Non-separated / Separated / Incidental / Accessory)
- Type of Construction + # Stories
- Sprinklered: Full / Partial / None — **plus System Type: 13 / 13R / 13D** (this differs from Building PRR — must specify NFPA standard)
- Monitored Y/N, Unlimited Area Y/N, High Rise Y/N

## §B — General Requirements

| PRR line | Where to verify |
|----------|----------------|
| Coversheet present or embedded; clear scope | A0.0 + general notes |
| **"I am familiar with FFX Common Plan Review Comments + plan reviewer contact info"** | KCY peer reviewer self-affirms (always Complies) |
| Drawings reference all applicable codes | G0.x code analysis |
| Industrialized Building Affidavit | Submittal package — N/A unless industrialized building |
| Designer contact on drawings | Title block per sheet |
| Sealed by VA RDP | Title block / cover |
| Min scale 1/8"=1' + 1/4"=1' details | Sheet review |
| Graphic scale + grid IDs | Each sheet |
| **Plans "For Construction"** — NOT "For Bid Only" / "Permit Only" / "<100% Progress" | Sheet titles + cover — auto-reject if any of those phrases appear |
| Key plan on each applicable drawing | Each sheet |
| True north on floor + site plans | Each sheet |
| Building + trades plans oriented same plan north (or marked exception) | Cross-check Arch vs MEP vs FP |
| Fire-rated walls delineated on Arch + MEP + Fire + IT plans w/ design # + legend | Cross-check across disciplines |
| Revisions: marked on Index, cover letter, revision clouds + delta + # + date | Resubmittal only |

## §C — Phased Construction

Mark **N/A for the entire section** if not phased: "Single-phase construction."

If phased, mark each line. Write "See Phasing Plan PH-XXX" reference if helpful.

---

## §D — Architectural Plans (FP cross-check)

For each line, verify the architectural plans show what the FP reviewer needs:
- Code analysis narrative — verify chapters cited match scope
- Height/area/grade plane calc — same as Building §C
- Construction Type matrix — must include all elements listed in `02b §D3`
- Use group + occupancy classification + mixed-use calcs
- Plans, areas, room names, OL calc by function
- Furniture plans
- Partition layout with UL #
- **Life Safety plan FS-XXX per floor** — biggest deficiency source; verify all 13 content items in `02b §D10`
- Elevations + sections + ramps + stairs + finishes
- **Door schedule + hardware sets per door** (no "Existing to Remain" / "Refer to Specs")
- Penetration details + firestop/draftstop notes + partition schedule with UL #

---

## §E — Architectural Site Information

If small interior alteration with no site impact → mark this section N/A: "No site work; interior alteration only."
Otherwise verify each line on civil + arch site sheet.

---

## §F — Fire Protection / Life Safety (longest section, most deficiencies)

For each line:
- ☑ Complies if drawings + submittal documents demonstrate the requirement
- ☐ N/A only if scope doesn't trigger (e.g., "no smoke control system" → smoke-control lines N/A)
- Deficiency → "See Deficiency Log #X"

**Smoke Control specific (Kyle's meeting takeaway):**
- "Smoke control record has been submitted..." → check that **the record number is on the plans**. If missing, this is a deficiency. The submittal pipeline (Steps 1-3) must be tracked outside the PRR — note status in Comments if helpful.
- "Smoke Control Manual + supplemental checklist reviewed" → KCY peer reviewer self-affirms.

**Door operation hardware:**
- The PRR line about door hardware schedule maps to `02b §D16`. Verify hardware sets, access control hardware, special locking devices, etc.

**In-building emergency communications:**
- High-rise → wired phones per §907.2.13.2 + §918
- Other → documentation showing no impedance (§918.1 Exception 5) OR infrastructure per §918.1.1

---

## §G / §H / §I / §J — MEP + Telecom cross-check

These sections are short — usually 4-6 lines each. They check:
- Rated/smoke walls shown with design #
- Room names + grid lines match Arch
- Firestop/penetration details with design #
- Discipline-specific FP touchpoints (smoke detection, dampers, exit signs, illumination)

For each: cross-reference back to that discipline's drawings.

If you, KCY, are also doing a separate full Mechanical / Plumbing / Electrical peer review for the same project, the deeper review goes in those checklists (`04`, `05`, `06`); this FP PRR section is just verifying that the FP-relevant items are on the discipline drawings.

---

## Comments column policy (FP-specific)

- ✅ Comments OK to point to deficiency log row
- ✅ N/A justifications: scope-level only (e.g., "No flammable/combustible tank in project scope")
- ❌ "Designer should add..." — no
- ❌ "Recommend..." — no
