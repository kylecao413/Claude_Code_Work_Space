- [ ] Code edition: 2021 VCC (new) / 2021 VEBC (existing/alteration)
- [ ] Occupancy/Group(s) — existing AND proposed if change of occupancy
- [ ] Type of Construction (VCC §601)
- [ ] Sprinklered (Full / Partial / None) + Monitored Y/N
- [ ] High Rise Y/N, Critical Structure Y/N, Atrium Y/N, Unlimited Area Y/N, Mixed Use type
- [ ] Project type: New Building / Addition / Level 1 Alt / Level 2 Alt / Level 3 Alt / Repair / Change of Occupancy / Moved / Historic
- [ ] Parcel-level: zoning case #, building-related proffers, Department of Code Compliance case Y/N

> **Why this is step 0:** The Plan Review Record asks for all of this on page 1; the Coversheet must mirror it. If any field is missing on the architect's plans, that's already a rejection comment. Capture it once, reuse it across all 5 trade checklists.

### G6 Sub-checklist — Door details Fairfax cites most often (Kyle's meeting takeaway #2)

- [ ] Manual flush bolts NOT used on means of egress doors (auto flush bolts only on inactive leaf of pairs) (§1010.1.9.4, §1010.1.9.3(3))
- [ ] Key-operated locks on main exterior B/F/M/S doors meet ALL 3 items of §1010.1.9.3(2)
- [ ] Door **clear width** ≥ 32" measured between face of door and stop at 90° (§1010.1.1)
- [ ] Doors side-hinged + swing in direction of egress where OL ≥ 50 (§1010.1.2.1)
- [ ] Door hardware operable without tight grasping/pinching, 34"–48" AFF (§1010.1.9.1, §1010.1.9.2)
- [ ] Unlatching ≤ 1 operation (§1010.1.9.5)
- [ ] Panic / fire exit hardware where required: A or E with OL ≥ 50, H, electrical rooms ≥ 1200 A (§1010.1.10)
- [ ] Fire doors self-closing + latching per VCC §716.5.9 + NFPA 80
- [ ] **Sensor-release egress doors** comply with §1010.1.9.8 — card reader + motion sensor + manual unlock within 5' shown on plan; tied to fire alarm; max 2 such doors per egress path
- [ ] **Delayed egress** per §1010.1.9.7 — sprinklered building only (except A/E/H); 6 items listed in §1010.1.9.7 noted on plan; ≤ 1 delayed door per egress path
- [ ] **Stairway doors lockable** only if they meet exception in §1010.1.9.11
- [ ] Doors in series have 48" clear between (§1010.1.8); turnstiles per §1010.3
- [ ] Door + hardware projection ≤ 7" into required egress width when fully open (§1005.7.1)
- [ ] Landings on each side of door, ≥ 44" depth, ≥ door width (§1010.1.5, §1010.1.6)

## M. Top-15 Building rejection patterns to pre-empt (from `rej_building.pdf`)

Sorted by frequency we expect across DC-area commercial peer reviews:

1. Coversheet missing or incomplete → A1
2. Door schedule + hardware schedule missing → G7, G8 (also blocks all egress + accessibility review)
3. Statement of Special Inspections missing/incomplete → A3, I11
4. Accessible route passes through prohibited spaces → H1
5. Common path / travel distance exceeded → G14
6. Corridor dead-end > limit OR continuity broken → G15
7. UL design numbers missing for fire-rated assemblies → E10
8. Structural design loads (snow/wind/seismic/live) not all listed with Fairfax-specific values → I1–I5
9. Geotech report missing where required (>2,000 psf assumed or any deep foundation) → B3
10. Vestibule missing on non-exempt entrance → J3
11. Energy R-values don't match VECC C402.1.3 → J2
12. Egress illumination not shown at exit discharge → G4
13. Type B dwelling unit accessibility not detailed → H4
14. ICC-ES report missing for EIFS / proprietary materials → A5
15. Building-related proffer not incorporated → A14

When scanning a new submittal, **start by binary-checking 1–4 in the first 10 minutes** — if any are missing, the rest of the review is moot until they're fixed.

- Observation describes WHAT is non-compliant on the drawings + cites VCC/VEBC/ANSI section.
- **No suggested correction language.** That belongs to the design professional. Suggesting fixes = unauthorized practice + conflict of interest as peer reviewer.
- No internal cross-reference codes (PRR §, our pattern numbers).
